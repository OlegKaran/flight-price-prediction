# ✈️ Предсказание цен на авиабилеты

Веб-приложение, которое прогнозирует стоимость авиабилета по параметрам рейса: маршрут, дата вылета, количество пересадок, насколько заранее покупается билет. В основе — модель машинного обучения, обученная на реальных ценах с Aviasales.

## Работа с данными

### Сбор данных

Данные собираются собственным парсером [`parser.py`](parser.py) через API Travelpayouts (Aviasales), эндпоинт `month-matrix`. Парсер перебирает **все пары из 18 городов** и цены по каждому маршруту на **12 месяцев вперёд**. Предусмотрены повторные попытки при обрывах связи и обработка лимита запросов (ошибка 429). Результат сохраняется в единый датасет.

### Очистка

- Удаляются колонки, не несущие полезной информации, такие как `gate`, `return_date`, `trip_class`, `show_to_affiliates`, `actual`.
- Отбрасываются строки с нулевыми `duration` и `distance` — это явные артефакты сбора.

### Предрасчёт маршрутов (`routes_data.json`)

Расстояние и длительность перелёта — важные признаки для модели, но пользователь их не вводит. Эти параметры для каждого маршрута заранее взяты из датасета и сохранены в `routes_data.json`. На инференсе сервис просто берёт готовые значения по ключу маршрута.

### Предобработка признаков 

Обработка признаков была сначала проведена в ноутбуке, после чего вся логика была вынесена в один класс `FlightPricePredictor` ([`ml_service.py`](ml_service.py)):

- Числовые признаки (`duration`, `distance`, `days_to_departure`) масштабируются `StandardScaler`.
- Категориальные признаки кодируются `OneHotEncoder` (с `drop='first'` и строгой обработкой неизвестных категорий).
- Обученные `StandardScaler` и `OneHotEncoder` были сохранены одним общим файлом `preprocessor.pkl`

`ml_service.py` подключается и к FastAPI, и к Streamlit, так что оба интерфейса обрабатывают запрос абсолютно одинаково.

## ➕ Добавленные признаки


| Признак | Как считается | Что показывает |
|---|---|---|
| `days_to_departure` | `depart_date − found_at` | За сколько дней до вылета ищется билет (глубина покупки) |
| `departure_month` | месяц из даты вылета | Сезонность спроса по месяцам |
| `departure_day_of_week` | день недели вылета | Разница будни / выходные |
| `is_weekend` | день недели ≥ 5 | Флаг вылета в выходной |
| `season` | группировка месяца | Winter / Spring / Summer / Autumn |
| `month_part` | часть месяца по дню | Начало / середина / конец месяца |
| `departure_year` | год вылета − 2026 | Год относительно точки отсчёта |

## 📊 Метрики моделей

За baseline была взята линейная регрессия с L2-регуляризацией (`Ridge`). Затем через `RandomizedSearchCV` перебирались ансамблевые модели с логированием в MLflow.

| Модель                            | MAE, ₽ |  MAPE  |
|-----------------------------------|:------:|:------:|
| Ridge (baseline)                  |  4293  | 29.05% |
| RandomForest                      |  2304  | 13.77% |
| **XGBoost** (Лучшая модель на CV) |  1743  | 9.60%  |
| LightGBM                          |  1820  | 10.22% |
| CatBoost                          |  1892  | 10.92% |

Итог: лучшая модель XGBoost снижает среднюю ошибку почти на **60%** относительно baseline (MAE 4293 → ~1740 ₽).

> ℹ️ Параметры лучшей модели XGBoost: `n_estimators=900`, `learning_rate=0.1`, `max_depth=8`.
> Все эксперименты (гиперпараметры, MAE,  артефакты моделей) логировались в **MLflow** с локальной SQLite-базой.

## 🧩 Как устроен сервис

Запрос пользователя проходит один и тот же путь в обоих интерфейсах:

1. Пользователь выбирает **города, дату вылета и число пересадок**.
2. По маршруту из `routes_data.json` подтягиваются `distance` и `duration`.
3. `FlightPricePredictor.prepare_data` генерирует признаки и применяет сохранённые `scaler` и `encoder`.
4. XGBoost выдаёт прогноз цены в рублях.

## 📡 Пример запроса

Эндпоинт **`POST /predict`** принимает параметры рейса и возвращает предсказанную цену.

> 📌 Города указываются **русскими названиями** (например, «Москва», «Барселона») — сервис сам переводит их в IATA-коды. Список доступных городов задан в `config.py`.

```bash
curl -X POST "http://localhost:8000/predict" \
     -H "Content-Type: application/json" \
     -d '{
       "depart_date": "2026-08-15",
       "origin": "Москва",
       "destination": "Барселона",
       "number_of_changes": 0
     }'
```

**Ответ:**

```json
{
  "found_at": "2026-07-14",
  "depart_date": "2026-08-15",
  "route": "Москва (MOW) -> Барселона (BCN)",
  "number_of_changes": 0,
  "predicted_price": 18500,
  "prediction_time_ms": "2.315",
  "currency": "RUB"
}
```

Если маршрут отсутствует в базе, сервис вернёт понятную ошибку `404` с предложением выбрать другой маршрут.

## 🛠 Стек технологий

- **Машинное обучение:** XGBoost (финальная модель), CatBoost, RandomForest, LightGBM, scikit-learn
- **Трекинг экспериментов:** MLflow + SQLite
- **Backend / REST API:** FastAPI, Uvicorn, Pydantic
- **Frontend:** Streamlit
- **Данные:** Pandas, Joblib (сериализация препроцессоров), Requests (сбор данных)
